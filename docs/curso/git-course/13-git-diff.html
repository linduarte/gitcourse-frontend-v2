Comparando versões

Em um repositório Git, é comum existirem várias versões de um mesmo arquivo ao longo do tempo. Cada versão registrada é chamada de commit. Um commit representa um ponto específico da evolução do arquivo dentro do repositório.

  • Diretório de trabalho — versão em edição.
  • Versão staged — adicionada ao index via git add.
  • Versão comitada — registrada no histórico.

No mundo real, sem Git, você poderia ter várias cópias de um arquivo no diretório de trabalho. Com o Git, cada versão é registrada como um commit, mantendo tudo organizado.

Agora vamos conhecer um comando essencial para visualizar diferenças entre versões: o git diff. 

git diff

Para iniciarmos o uso do git diff, é importante saber o status do repositório após o commit da letra “D”.

Vamos implementar uma nova feature request adicionando a letra “E” aos arquivos pilots.html, cities.html e names.html.

Após as alterações, adicionamos os arquivos ao index usando git add.

Em seguida, adicionamos a letra “F” aos mesmos arquivos.

A imagem mostra que a letra “F” ainda não foi adicionada ao index, enquanto a letra “E” já está staged.

Temos então três situações distintas:
– letra D comitada
– letra E staged
– letra F apenas no diretório de trabalho

Git diff — Como comparar mudanças

O comando git diff é extremamente útil para visualizar diferenças entre estados da base de código.

Ele permite comparar:

  • Diretório de trabalho vs. último commit
  • Staged vs. último commit
  • Dois commits específicos
  • Dois branches
  • Arquivos específicos

Comandos git diff por arquivo

git diff HEAD file_name
git diff file_name
git diff --staged file_name
git diff branch1 branch2 file_name
git diff commit1 commit2 file_name

Usaremos o arquivo pilots.html para demonstração.

A seguir interpretaremos a saída do comando git diff pilots.html.

Comando executado

  • git diff
  • Compara a versão atual com a última comitada.

Primeiras linhas

  • diff --git a/pilots.html b/pilots.html
  • Indica que o arquivo está sendo comparado.

Índice

  • index fb4318f..32706f8 100644
  • fb4318f → hash anterior
  • 32706f8 → hash atual
  • 100644 → permissões do arquivo

Identificação das versões

  • --- a/pilots.html
  • +++ b/pilots.html

Bloco de contexto

  • @@ -22,6 +22,7 @@
  • Mostra onde ocorreu a mudança.

Conteúdo antes e depois

Estado do repositório

  • A linha Delta já estava comitada.
  • A linha Foxtrot está unstaged.
  • A letra E está staged.

Testando no repositório

Para ver diferenças staged:

git diff --staged pilots.html

Alterações desde o último commit

git diff HEAD pilots.html

Comparando dois commits

git diff HEAD~1 HEAD pilots.html

Obtendo hashes pelo git log

git log --oneline --decorate
git diff d298f74 2dded79 pilots.html

Commit das pendências

O commit da letra F foi precedido de git add.
